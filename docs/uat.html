User Acceptance Testing — Kandank Iwak

UAT berikut dirancang dari sudut pandang pengguna akhir: Pelanggan (publik) dan Admin/Pemilik. Setiap use case utama memiliki minimal satu skenario pengujian.

No Skenario Pengujian Role Fitur yang Diuji Langkah Pengujian Ekspektasi Hasil Status
1 Pelanggan ingin menemukan nila segar di bawah Rp 40.000 Pelanggan Melihat Katalog Produk Buka beranda → klik "Lihat Katalog" → pilih kategori "Ikan Segar" → geser slider harga ≤ 40000 Daftar produk tersaring sesuai kriteria, jumlah produk yang ditampilkan terlihat di pojok atas Pass
2 Pelanggan ingin tahu detail Nila Segar Hidup sebelum membeli Pelanggan Melihat Detail Produk Klik kartu produk → baca deskripsi, harga, stok → atur jumlah 2 kg Subtotal otomatis menjadi 2 × harga, tombol "Tambah ke Keranjang" aktif Pass
3 Pelanggan menambah 2 produk berbeda lalu ubah qty di keranjang Pelanggan Mengelola Keranjang Belanja Tambah Nila Segar 1 kg + Bumbu Pecak 2 pack → buka /cart → ubah qty Nila menjadi 3 → hapus Bumbu Subtotal & total ter-update, item Bumbu hilang, keranjang persist saat reload Pass

Tangerang Pelanggan Checkout & Membuat Pesanan Isi nama, no WA "8123456789", pilih kecamatan "Cikupa" → kelurahan "Pasirjaya" → isi detail alamat → pilih BCA → klik Lanjutkan Halaman /checkout/success terbuka dengan kode KI-YYYYMMDD-XXXX dan instruksi transfer Pass
5 Pelanggan konfirmasi pembayaran via WhatsApp setelah transfer Pelanggan Konfirmasi Pembayaran via WhatsApp Salin nomor rekening & nominal → transfer di m-banking → kembali ke halaman sukses → klik "Konfirmasi via WhatsApp" Aplikasi WhatsApp terbuka dengan pesan otomatis berisi kode pesanan, item, dan total Pass
6 Pelanggan mengecek status pesanan setelah dibayar Pelanggan Lacak Pesanan Buka /track → masukkan kode KI-20260606-AB12 → klik Lacak Stepper status muncul (Dipesan/Dibayar/Dikirim/Selesai) sesuai posisi saat ini, ringkasan item tampil Pass
7 Admin mengakses dashboard untuk memantau toko Admin / Pemilik Login Admin Buka /admin/login → isi email & password Firebase Auth → klik Masuk Dashboard /admin tampil dengan kartu statistik (penjualan bulan ini, total pesanan, produk aktif) Pass
8 Admin menambah produk baru "Nila Frozen Pack" Admin / Pemilik Mengelola Produk Buka /admin/products → klik Tambah → upload gambar → isi nama "Nila Frozen Pack", harga 42000, stok 30 → Simpan Toast "Produk ditambahkan", produk tampil di daftar admin dan langsung muncul di /products Pass
9 Admin memajukan status pesanan ke "Dibayar" Admin / Pemilik Mengelola Pesanan Buka /admin/orders → filter "Dipesan" → klik "Tandai Dibayar" pada baris pesanan → konfirmasi Badge berubah ke "Dibayar", pelanggan melihat update di halaman /track Pass
10 Admin membatalkan pesanan yang salah Admin / Pemilik Mengelola Pesanan (Membatalkan) Buka /admin/orders → klik "Batalkan" pada pesanan ordered → konfirmasi "Ya, Batalkan" Status menjadi "Dibatalkan" merah, pelanggan melihat banner pembatalan di /track Pass
11 Admin menambah metode pembayaran DANA Admin / Pemilik Mengelola Metode Pembayaran Buka /admin/payment-methods → klik Tambah → pilih DANA → isi atas nama & nomor → Simpan DANA tampil di tabel admin dan tersedia sebagai pilihan di halaman checkout pelanggan Pass
12 Admin menjalankan analisis Apriori untuk menemukan kombinasi favorit Admin / Pemilik Menjalankan Analisis Apriori Buka /admin/apriori → atur Min Support 10%, Min Confidence 30% → klik Jalankan Analisis Tabel Association Rules tampil terurut confidence desc, hasil ter-simpan dan dipakai widget "Sering Dibeli Bersama" di toko Pass
