Desain Database NoSQL — Kandank Iwak Firestore v1

Dokumen ini mendeskripsikan struktur basis data Firebase Firestore yang digunakan oleh aplikasi Kandank Iwak — toko ikan nila segar dengan fitur dashboard admin dan analisis Apriori untuk widget "Sering Dibeli Bersama".

1. Strategi Pemodelan NoSQL

Aplikasi ini menggunakan kombinasi Embedding dan Referencing sesuai karakter setiap koleksi:

  • Embedding dipakai pada koleksi orders untuk field items, address, dan payment_method. Snapshot harga & nama produk disimpan langsung di dalam dokumen pesanan agar riwayat tetap akurat sekalipun produk asli diubah/dinonaktifkan di kemudian hari (lihat src/lib/firebase/orders.js fungsi addOrderAndTransaction).
  • Referencing dipakai pada koleksi transactions: field items hanya menyimpan daftar product_id (string array). Bentuk ini sengaja minimalis karena transactions menjadi sumber data algoritma Apriori — hanya butuh keanggotaan item per transaksi, bukan harga.
  • Document ID khusus dipakai pada orders: ID dokumen sama dengan kode pesanan KI-YYYYMMDD-XXXX sehingga pelanggan bisa getDoc langsung saat melacak pesanan tanpa query.
  • Koleksi terpisah (products, payment_methods, rules) untuk entitas yang sering dibaca independen oleh halaman toko & dashboard.

Pilihan ini sengaja menghindari sub-koleksi karena seluruh akses transaksional (order + transaction) dijamin atomik melalui writeBatch Firestore.

2. Struktur Koleksi

Tree berikut merefleksikan koleksi yang dideklarasikan di src/lib/firebase/collections.js:

  • 📁 products (doc id: auto)
    • 📄 QkW0aB2xz3 → name, category, price, stock, unit, description, image_url, active, created_at
  • 📁 orders (doc id: kode pesanan)
    • 📄 KI-20260606-AB12 → customer_name, contact, total, status, notes, created_at
      • 🧩 items[] (embedded): product_id, name, price, qty
      • 🧩 address (embedded): kecamatan_code, kecamatan_name, kelurahan_code, kelurahan_name, detail
      • 🧩 payment_method (embedded snapshot): id, payment_method, account_name, account_number
  • 📁 transactions (doc id: auto)
    • 📄 7iV3... → order_code (ref orders), items: string[], source, date
  • 📁 payment_methods (doc id: auto)
    • 📄 pmBcaXyz → payment_method ("bca"), account_name, account_number, created_at
  • 📁 rules (doc id: auto, di-replace saat analisis)
    • 📄 rule_001 → antecedent[], consequent[], support, confidence, parameters, calculated_at
  • 📁 admins (opsional)
    • 📄 uid Firebase Auth → email, role
